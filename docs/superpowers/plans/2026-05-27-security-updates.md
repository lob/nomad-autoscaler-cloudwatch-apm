No plugin source changes required, only `go.mod`/`go.sum` and toolchain config files.

**Tech Stack:** Go 1.25, `golang.org/x/net`, `golang.org/x/sys`, `golang.org/x/text`, `hashicorp/nomad-autoscaler v0.5.0`, `google.golang.org/grpc v1.81.1` (transitive via nomad-autoscaler), `govulncheck`

- [ ] **Step 2: Write the failing test first**

Add this test to `plugin/plugin_test.go` (append after `TestApmPlugin_PluginInfo`):

```go
// TestNoSymbolVulnerabilities shells out to govulncheck and fails if any
// Symbol-level CVEs are reachable from this binary. Symbol-level means the
// vulnerable function is in the actual call graph — not just present in a dep.
func TestNoSymbolVulnerabilities(t *testing.T) {
	govulncheck, err := exec.LookPath("govulncheck")
	if err != nil {
		t.Skip("govulncheck not in PATH — install with: go install golang.org/x/vuln/cmd/govulncheck@latest")
	}

	cmd := exec.Command(govulncheck, "-json", "./...")
	cmd.Dir = filepath.Join("..", ) // run from module root
	out, _ := cmd.Output()          // non-zero exit is expected when vulns found

	// Parse the JSON output looking for Symbol-level findings.
	type finding struct {
		Finding struct {
			OSV   string `json:"osv"`
			Trace []struct {
				Function string `json:"function"`
			} `json:"trace"`
		} `json:"finding"`
	}

	var symbolVulns []string
	for _, line := range bytes.Split(out, []byte("\n")) {
		if len(line) == 0 {
			continue
		}
		var f finding
		if err := json.Unmarshal(line, &f); err != nil {
			continue
		}
		// A non-empty Trace means the symbol is reachable.
		if f.Finding.OSV != "" && len(f.Finding.Trace) > 0 {
			symbolVulns = append(symbolVulns, f.Finding.OSV)
		}
	}

	assert.Empty(t, symbolVulns,
		"Symbol-level CVEs found in call graph — upgrade the affected dependencies: %v", symbolVulns)
}
```
